Appendix B: Tools

Using the Firefox Developer tools

Open the Developer tools in Firefox by using the ‘hamburger’ menu (≡) near the top right of the window and choosing Web Developer and then Toggle Tools (or, using Chrome, choose the ‘hamburger’ menu (three vertical dots), then More tools, then Developer tools). For Firefox and Chrome, a shortcut on Windows PCs is to use Ctrl-Shift-I. This will open a window at the bottom (or side) of the browser window which can be used to inspect elements on the page by selecting the Inspector tab or the small symbol to the left of this tab. The box model can be shown (in Firefox by selecting the 'Layout' tab on the right hand side of this window), so that you can see the size of margins and padding for each element. You can also use it to examine the CSS rules applied to a specific element by highlighting the relevant element and then selecting the tab on the right hand side that says ‘rules’. The inspector comes in very handy when the complexity of the HTML and CSS files increases. You will find yourself using it a lot in future if your website layout is not doing what you expect it to be doing – you can inspect individual elements to check which rules have been applied to them. It will also come in useful when you do some JavaScript when the console and debugger tabs become more useful. The Developer tools are very useful and you should practise using them regularly – see https://developer.mozilla.org/en-US/docs/Tools/Tools_Toolbox for further details. (You might also want to take a look at the Firefox Developer Edition and its tools: https://www.mozilla.org/en-US/firefox/developer/.)

Inspecting the source code of web pages

It is possible to examine the source code of any website you visit by right clicking on the browser display area and selecting the option to ‘View Page Source’. This can be useful if you want to examine how a particular effect has been achieved on a web page that you visit. It is not, however, an excuse for plagiarism.

Markup validation

When you are developing a website it is a good idea to validate your code. (Note: If everyone in the class uses the validators listed below at a similar time whilst using a particular computer lab, it is likely that the particular subnet of the University network will be blocked temporarily by the validation service. This won't happen if you are running on your own computer at home.)

HTML: W3C host a validator for HTML: http://validator.w3.org/. Validation can be done by uploading the relevant file (or by URL if the file resides on a web server). There are ‘More Options’ to set the document type used.

Exercise: You should validate one of the html files you have been working with.

CSS: Likewise, you can validate your css file at the W3C’s CSS validation service: http://www.css-validator.org/.

Exercise: You should validate one of the css files you have been working with.
