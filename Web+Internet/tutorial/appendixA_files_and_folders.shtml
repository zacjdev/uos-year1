Appendix A: Folders and files

You can organise your files into a hierarchy of folders (or directories - some people refer to them as directories and some as folders. I'll switch between the two). Figure 1 gives a graphical depiction of a hypothetical folder hierarchy. In this example, the U: drive is called the root of a userâ€™s folder hierarchy (but, since there are many users, this is just one level in a larger network file system). For the particular user, the file a.txt has the full pathname of U:\other\a.txt2. You can create such folder hierarchies in a number of ways. They can be done using a graphical interface tool such as Windows Explorer (or an equivalent if you are running Linux or MacOS) or from a command line window. It is a good idea to create a folder hierarchy to organise your work.

Note: U:\other\a.txt2. Forward slash or backslash? On Windows systems a backslash is used as part of a file system pathname. For all other systems, it is a forward slash.
